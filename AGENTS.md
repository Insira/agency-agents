# The Agency: Codebase Documentation

This document serves as a technical reference for the structure, architecture, and workflows of **The Agency** repository.

## 🎭 Overview
**The Agency** is a centralized repository for high-quality, personality-driven AI agent definitions. It uses a specialized build-and-deploy pipeline to distribute these agents across various AI coding assistants and CLI tools (e.g., Claude Code, GitHub Copilot, Cursor, Aider).

## 🏗️ Repository Architecture

The repository is organized into three hierarchical layers:

### 1. Source Layer (The Source of Truth)
All agents reside in the root directory, organized by domain (e.g., `engineering/`, `design/`, `marketing/`).
- **Format**: Individual Markdown (`.md`) files.
- **Structure**: Each file uses **YAML frontmatter** for metadata (name, description, model, etc.) and a **Markdown body** for the agent's persona, mission, and workflows.

### 2. Transformation Layer (The Compiler)
The `scripts/convert.sh` script acts as a compiler.
- **Function**: It reads the source `.md` files and transforms them into tool-specific formats.
- **Output**: The transformed files are stored in the `integrations/` directory.
- **Examples**:
    - Converting agents to `.mdc` for Cursor.
    - Aggregating agents into a single `CONVENTIONS.md` for Aider.
    - Generating `SKILL.md` for Antigravity.

### 3. Deployment Layer (The Installer)
The `scripts/install.sh` script acts as the deployment engine.
- **Function**: It automates the installation of converted agents into the user's local environment.
- **Features**:
    - **Auto-detection**: Scans the system for installed tools (Claude Code, Copint, etc.).
    - **Interactive UI**: Allows users to select which tools to populate via a terminal UI.
    - **Local Scoping**: Supports both user-wide (e.g., `~/.claude/agents/`) and project-scoped (e.g., `.cursor/rules/`) installations.

## 🛠️ Developer Workflows

### Adding a New Agent
1. Create a new `.md` file in the relevant domain directory.
2. Define the YAML frontmatter (mandatory: `name`, `description`).
3. Define the agent's personality and mission in the Markdown body.
4. Run `./scripts/convert.sh` to update the `integrations/` directory.

### Adding Support for a New AI Tool
1. **Update `scripts/convert.sh`**: Implement a new `convert_<toolname>` function that handles the specific required format.
2. **Update `scripts/install.sh`**: 
    - Implement a `install_<toolname>` function.
    - Add the tool to the `ALL_TOOLS` array.
    - Add detection logic (e.g., `detect_<toolname}`).
    - Update the UI/labeling logic.

## 📂 Key Directories
- `[domain]/`: Contains the master agent definition files.
- `integrations/`: Contains the tool-specific files generated by `convert.sh`.
- `scripts/`: Contains the core logic for conversion and installation.
- `examples/`: Contains usage scenarios and case studies.
